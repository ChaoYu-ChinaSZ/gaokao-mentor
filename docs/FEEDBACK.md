# 用户反馈收集方案

## 一、设计原则

1. **零门槛** — 反馈成本要极低，用户随手就能操作
2. **不打断体验** — 不能在对话中途弹窗问反馈
3. **分类收集** — 技术问题、体验问题、结果准确性要分开
4. **闭环追踪** — 每条反馈有回应，重大问题通知用户改进

---

## 二、反馈收集渠道矩阵

### 渠道1：对话结束时的轻量反馈（核心）

在对话完成 OUTPUT 状态后、END 之前，自动插入轻量反馈：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  反馈（选填，30秒完成）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

这次对话，你感觉：

① 有帮助 ② 一般 ③ 没用

（直接回复数字即可）
```

收到反馈后存储到用户档案：

```python
profile.post_conversation_rating = 3  # 1/2/3
```

**数据用途：** 追踪整体满意度，识别流程问题

---

### 渠道2：深度反馈（可选填）

对话结束后，发起可选的深度反馈：

```
你愿意花2分钟填写详细反馈吗？帮助我们优化这个工具。

回复"详细反馈"开始，或直接关闭对话。
```

用户回复"详细反馈"后，进入5问：

```
1. 推荐的专业方向，你之前了解过吗？
   A. 完全没听说过  B. 听说过但没深入了解  C. 了解过
2. 推荐结果和你对自己的认知相比：
   A. 非常符合  B. 部分符合  C. 不太符合  D. 完全不符
3. 对话中哪个部分最有价值？（单选）
   A. Ikigai四问  B. 现实约束探测  C. RIASEC测评  D. 最终推荐
4. 觉得哪里可以更好？（简答）
5. 你身边有需要高考咨询的朋友吗？
   A. 有  B. 没有
```

收集后存储到档案并打印确认。

---

### 渠道3：GitHub Discussions（技术用户反馈）

**入口：** GitHub仓库主页 → Discussions

预设分类：
- 🐛 BUG反馈 — 技术问题、报错
- 💡 功能建议 — 希望增加的功能
- 🎓 案例分享 — 使用体验、结果分享
- ❓ 使用问题 — 部署问题、配置问题

---

### 渠道4：微信内置反馈（普通用户）

用户在任何时候输入"反馈"或"提建议"，立即触发反馈收集：

```
请告诉我你的反馈（直接打字发过来即可）：
- 哪里不好用？
- 哪里可以更好？
- 你希望增加什么功能？

收到后会认真处理，谢谢你。
```

内容存储到：`~/.hermes/gaokao_profiles/{openid}_feedback.txt`

---

### 渠道5：微信匿名反馈（二维码）

在推广时附上匿名反馈二维码（指向一个简单的表单链接）：

- 表单平台：腾讯问卷 / 问卷星（免费额度够用）
- 问卷内容：5道题，2分钟完成
- 匿名提交，无需登录

二维码放在GitHub README底部。

## 三、反馈数据存储结构

用户档案扩展字段：

```python
@dataclass
class FeedbackRecord:
    openid: str
    timestamp: str          # ISO格式时间
    rating: int             # 1/2/3
    recommended_majors_match: str   # A/B/C/D（深度反馈）
    valuable_section: str   # A/B/C/D（深度反馈）
    free_feedback: str      # 用户自由文字
    conversation_id: str    # 对应哪个session
    skill_version: str     # 当前skill版本
```

反馈日志：`~/.hermes/gaokao_feedback/YYYY-MM.json`

---

## 四、反馈处理流程

```
用户提交反馈
    ↓
存储到 gaokao_feedback/
    ↓
每月1次汇总分析
    ↓
识别Top3问题
    ↓
制定优化方案
    ↓
在GitHub Discussions公布改进
    ↓
下一版本更新时通知用户
```

```markdown

### Top问题

1. [问题描述] — [出现次数]次 — [优先级：高/中/低]

3. [问题描述] — [出现次数]次 — [优先级]

### 功能建议

1. [建议内容] — [提出人]

### 改进计划

### 用户案例（已脱敏）

- [案例1描述]
- [案例2描述]
```

## 六、用户反馈激励

| 行为 | 激励 |
|---|---|
| 完成深度反馈 | 记录为"早期贡献者"，v2.0发布时邮件通知 |
| 在GitHub Discussions发帖 | 回复确认，采纳后@致谢 |
| 帮助发现BUG | 在更新日志中署名鸣谢 |
| 分享使用案例 | 脱敏后收入"用户故事"页面 |

---

## 七、反馈数据分析指标

每月追踪：

- **满意度趋势**：评分是否在提升
- **各环节流失率**：多少用户跑到IKIGAI_SUMMARY就中断
- **推荐匹配度**：深度反馈中"推荐结果符合度"分布
- **反馈响应率**：发反馈邀评后，实际完成的比例
- **净推荐值（NPS）**："有帮助"比例 - "没用"比例
