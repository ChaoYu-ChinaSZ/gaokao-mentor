# 推广运营方案

## 目标

让更多 Hermes Agent 用户发现、安装、使用 gaokao-mentor skill，
并通过收集真实用户反馈持续优化。

---

## 一、目标平台与策略

### 平台1：GitHub 本身（最核心）

**为什么：** 这是开发者发现工具的第一站，也是最精准的流量入口。

**具体动作：**

1. **完善 GitHub README**
   - 顶部 Banner：项目介绍 + 效果截图（对话截图）
   - Badge：Python 版本、License、Stars
   - 快速安装区（3行代码）
   - 贡献指南入口（CONTRIBUTING.md）

2. **开启 GitHub Discussions**
   - 用途：用户提问、反馈、分享使用案例
   - 预设分类：使用问题 / 功能建议 / 案例分享 / BUG反馈

3. **GitHub Actions 自动测试**
   - 每次 push 自动运行 `test_engine.py`
   - 确保 skill 始终可用

4. **Stars + Watch 增长**
   - 自己先宣传，让仓库有初始热度
   - Stars 数量是其他开发者判断项目价值的核心指标

---

### 平台2：知乎

**为什么：** 知乎是中国高质量内容搜索的入口。"高考志愿怎么填"、"高三学生如何选专业"等问题每年高考季都有极高搜索量。

**内容策略：**

文章标题方向：
- 《我用AI跑完了完整的志愿咨询，发现了一个被忽视的问题》
- 《高考志愿填报前，你做自我认知了吗？》
- 《高三学生最该问自己的四个问题（附AI对话体验）》

文章结构：
```
1. 痛点引入（高三学生选专业的迷茫）
2. 介绍传统方式的局限
3. 介绍自我认知工具（gaokao-mentor）
4. 展示真实对话体验（截图）
5. 给出GitHub链接+使用说明
6. 引导关注/收藏
```

发布时机：**高考出分前两周**（搜索量峰值）

---

### 平台3：小红书

**为什么：** 高三学生和年轻家长高度集中的内容社区，传播快，适合真实体验分享。

**内容策略：**

帖子结构（适合截图/短视频）：
- "我用AI做了高考志愿咨询，结果出乎意料"
- 截图关键对话节点
- 文末：GitHub链接 + "让更多高三学生用上"

短视频方向：
- 30秒演示：发"开始咨询" → 收到Ikigai第一问 → AI给推荐结果
- 配文："这个AI不问成绩，只问你自己"

---

### 平台4：即刻/知识星球

**为什么：** 聚集了大量对新工具敏感的早期用户和独立开发者。

**具体动作：**
- 在即刻发体验帖，附GitHub链接
- 加入高考/教育类知识星球，发软文介绍

---

### 平台5：微信志愿填报群

**为什么：** 精准用户池，家长和学生最集中。

**具体动作：**

潜伏策略：
1. 加入目标省份的"高考志愿填报群"
2. 前1-2周只分享价值内容（选专业的方法论）
3. 第三周开始分享："我做了一个AI工具，大家可以试试"
4. 发GitHub链接和体验说明

注意：不要一进群就发广告，容易被踢。

---

### 平台6：Reddit

**为什么：** 国际开发者社区，Hermes Agent 可能有国际用户。

**适合发帖的subreddit：**
- r/explainlikeimfive（教育相关）
- r/China（关于中国高考系统）
- r/SideProject（独立开发者分享）
- r/artificial（AI工具讨论）

内容：用英文写，介绍工具原理和使用方式。

---

### 平台7：Product Hunt

**为什么：** 产品发布平台，适合让更多独立开发者和工具爱好者发现。

发布时机：v2.0发布时（有完整功能后）

## 二、分阶段推广计划

### 阶段一：冷启动（现在 - 高考出分前1个月）

**目标：** 50 Stars，10个真实用户反馈

**动作：**
- [x] GitHub仓库初始化完成
- [ ] 完善GitHub README（Banner、Badge、截图）
- [ ] 开启GitHub Discussions
- [ ] 知乎发布第一篇推广文
- [ ] 即刻发体验帖
- [ ] 找3-5个种子用户完整跑完流程

**里程碑：** 知乎文章有100+收藏

---

### 阶段二：高考出分冲刺（出分前2周 - 出分后2周）

**目标：** 200 Stars，50个真实用户体验，10条有效反馈

**动作：**
- [ ] 小红书发布高考特辑内容（系列3-5篇）
- [ ] 知乎文章配合热点重发
- [ ] 在目标省份微信群定向推广
- [ ] 收集到的反馈整理，优化V2.0

**里程碑：** 日均UV > 100

---

### 阶段三：持续运营（高考后 - 下一届高考）

**目标：** 500 Stars，建立反馈闭环，形成口碑

**动作：**
- [ ] 根据反馈迭代V2.0
- [ ] 整理用户使用案例集（脱敏）
- [ ] 发布Product Hunt
- [ ] 建立用户社群（微信群/知识星球）
- [ ] 收集院校匹配需求（V2.0核心功能）

## 五、注意事项

1. **内容合规**：推广内容不承诺录取结果，不夸大效果
2. **隐私保护**：所有截图必须去除个人信息
3. **反馈驱动**：推广不是目的，收集反馈优化产品才是
4. **持续更新**：保持GitHub活跃度（每月至少1次commit）
